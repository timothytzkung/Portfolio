Ascend the Abyss - Processing (Java) Game

Java Processing Game Design & Development UML

A rogue-like, twin-stick shooter game created in Processing Java.

Client: University Project
Duration: 4 Weeks
Role: Game Designer & Developer
Completed: August 2022
Source Code

Project Overview

This project was inspired by my interest in rogue-like mechanics and games like The Binding of Isaac and Hades. I wanted to explore dungeon creation within Processing Java*, focusing on procedural generation, varied enemies, and item-based progression.

Players navigate procedurally generated rooms, battling enemies such as summoners and chargers while collecting items that enhance abilities like vision, speed, and damage.

Key Features

  • Each playthrough offers unique room layouts, ensuring replayability and dynamic exploration.
  • Enemies like summoners and chargers introduce varied combat challenges that test player strategy.
  • Players can collect items that enhance vision, movement speed, and damage, encouraging adaptive gameplay.
  • Handcrafted sprites and animations developed through iterative self-taught pixel art practice.
  • A focused scope that prioritizes polished dungeon generation, combat, and item systems over excessive feature bloat.
  • All aspects — design, programming, art, and testing — were handled individually, showcasing full-stack game development skills.

Challenges & Solutions

Challenges

I initially envisioned a large number of features, including complex dungeon layouts, numerous enemy types, and elaborate item effects. This made the project difficult to manage within my available time and resources.

Moreover, as someone with limited experience in pixel art, creating visually appealing sprites and animations proved difficult and time-consuming.

Lastly, Some item combinations made the game too easy, reducing the challenge and engagement for players.

Solutions

I narrowed the focus to core features — streamlined dungeon generation, a small but diverse enemy roster, and a few impactful items — which made the game more manageable and polished.

I taught myself pixel art iteratively, gradually improving asset quality through repeated practice and feedback, which resulted in a consistent and functional visual style for the game.

I conducted multiple rounds of playtesting with friends and adjusted item effects — toning down overpowered upgrades and ensuring each item provided a meaningful, balanced boost.

Technical Implementation

The project was built completely in Processing/Java. I used Object-Oriented Programming to structure my project, with game objects and elements managed by a custom GameManager.

All pixel-art asset creation, including sprite work and animation, was handled using Aseprite. Animations were handled using my own AnimationHandler to cycle through sprites based on the object's state.

Example Animation Component:

class Animation {
    PImage[] images;
    PVector size;
    int imageCount;
    int frame;
    int rate;
    
    Animation(String imagePrefix, int count, int rate, PVector size) {
        imageCount = count;
        images = new PImage[imageCount];
        this.size = size;
        
        this.rate = rate;
        frame = 0;
    
        for (int i = 0; i < imageCount; i++) {
        // Use nf() to number format 'i' into three digits
        String filename = imagePrefix + nf(i, 3) + ".png";
        images[i] = loadImage(filename);
        }
    }
    
    void display(float xpos, float ypos) {
        // Displays animated sprite
        if (frameCount % rate == 0) {
        frame = (frame+1) % imageCount;
        }
        imageMode(CENTER);
        image(images[frame], xpos, ypos, size.x, size.y);
    }
    
    int getWidth() {
        // Returns width of sprite
        return images[0].width;
    }
}

Lessons Learned

Managing scope creep during Ascend the Abyss taught me the importance of setting clear priorities and focusing on the essentials. By simplifying dungeon layouts, refining enemy designs, and narrowing item mechanics, I delivered a functional game while meeting deadlines.

In hindsight, I could have planned features more effectively by establishing milestones or sprints, which would have reduced last-minute adjustments. In future projects, I’ll focus on creating a more structured development timeline and allocating time for visual polish to elevate the overall quality of the experience.
